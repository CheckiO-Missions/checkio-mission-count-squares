Etant donné une séquence de points (coordonnées x, y sous forme d'entiers non négatifs), votre fonction doit compter combien de carrés existent de sorte que les quatre coins soient membres de points. Notez que ces carrés ne sont pas tenus d'être alignés sur l'axe, de sorte que leurs côtés doivent être horizontaux et verticaux. Par exemple, les points {% if is_js %}[0, 3], [3, 0], [6, 3], [3, 6]{% else %}(0, 3), (3, 0), (6, 3), (3, 6){% endif %} définissent un carré, même s'il peut ressembler à un losange depuis notre point de vue aligné sur l'axe.

D'ailleurs, si vous voulez essayer une autre approche et compter les carrés, en ayant des lignes entre les points, The Square Chest vous attend!

Pour identifier les quatre points qui constituent un carré, notez que chaque carré possède un point d'angle inférieur gauche x, y et un vecteur de direction dx, dy vers son point d'angle supérieur gauche qui satisfait à dx>= 0 & dy> 0, de sorte que les points {% if is_js %}[x+dx,y+dy], [x+dy,y-dx], [x+dx+dy,y-dx+dy]{% else %}(x+dx,y+dy), (x+dy,y-dx), (x+dx+dy,y-dx+dy){% endif %} pour les coins supérieur gauche, inférieur droit et supérieur droit de ce carré, respectivement, sont également inclus dans les points. Vous pouvez donc (effectivement) itérer à travers toutes les possibilités pour le point inférieur gauche et le vecteur de direction et être assuré de trouver tous les carrés de la grille.
